# アイキャッチ生成プロンプト（COLLAB-001 / 生前整理×美術品）

対象記事：「生前整理で出てきた絵画・美術品はどうする？価値の見極め方と後悔しない手放し方」
生成ツール：ChatGPT（GPT-4o 画像生成）想定

---

## ブランド・トーンの前提（既存アセットから抽出）
- 画風：**柔らかな水彩＋色鉛筆の絵本タッチ**（owl-character.png / og-image.png と統一）。
- 配色：**生成り・クリーム地**を基調に、**深い森の緑・セージグリーン**、**木の温かい茶色**、差し色に**控えめなテラコッタ**。淡く彩度低め。
- 空気感：暖かい朝の自然光、にじみのある優しい光、安心・希望・「寄り添い」。
- マスコット：丸眼鏡＋緑マフラーの茶色いフクロウ（頬は淡いピンク、にこやか）。
- NG：重く悲しい終活感、毒々しい色、写実的な写真調。

---

## ① 推奨プロンプト（フクロウ入り・そのままコピペ用）

```
柔らかな水彩と色鉛筆で描いた、温かみのある絵本風イラストを生成してください。日本の高齢者向け「生前整理」メディアの記事アイキャッチ画像です。横長16:9。

【シーン】明るい朝の光が差し込む、和やかな日本家屋のリビング。木のテーブルの上に、生前整理で出てきた美術品が丁寧に並んでいる——額装された風景画、くるりと巻かれた掛軸、小ぶりの陶磁器の壺と茶碗。やさしい手つきで一点の絵をそっと見つめる、品のよい年配の女性。

【マスコット】テーブルのそばに、丸い眼鏡をかけ深緑のマフラーを巻いた、ふんわり丸い茶色のフクロウが、小さな虫眼鏡を持って温かく見守っている（「価値をていねいに見極める」象徴）。頬は淡いピンク、目はにっこり閉じ、水彩タッチ。

【配色】生成り・クリーム色の背景を基調に、深い森の緑とセージグリーン、木の温かい茶色、差し色に控えめなテラコッタ。全体に淡く、彩度は抑えめで上品に。

【光・空気感】柔らかく暖かい朝日、ふんわりした光のにじみ、優しい陰影。安心感・希望・寄り添いの雰囲気。説明的すぎず、心がほどけるような穏やかさ。

【構図】被写体は画面の中央〜片側に寄せ、反対側（左上または右上）に後からタイトル文字を載せられる無地のクリーム色の余白を広めに確保すること。

【厳守】画像内に文字・ロゴ・透かしを一切入れない。写真のようなリアルさにしない。暗く重い雰囲気、ごちゃついた背景、原色の毒々しい色、手や指の崩れを避ける。
```

英語スタイル補助（必要なら末尾に追記）：
```
Style: soft Japanese watercolor + colored pencil, gentle picture-book illustration, warm and reassuring, muted cream/forest-green/terracotta palette, soft morning light, 16:9, generous empty negative space for a title overlay, NO text, NO logo, not photorealistic.
```

## ② 変種プロンプト（フクロウなし・上品な編集寄り）
マスコットをAIで描くと既存キャラと細部が一致しないため、「**背景シーンだけAI生成 → 後から実物の owl-character.png を合成**」したい場合はこちら。

```
（①の【マスコット】段落を削除し、以下に置換）
【主役】テーブルに並ぶ美術品（額装された風景画・巻いた掛軸・陶磁器の壺と茶碗）を、朝の光がやわらかく照らす静物的な構図。人物は手元だけ、または無人でも可。情緒は温かく、これから価値を見極めて大切に手放す前向きな空気。
```
※この場合、左右どちらかに owl-character.png を載せる前提で、その側を広めの無地余白にする。

## 生成AIスペシャリストからの実務メモ
- **日本語タイトルは画像に入れない**：GPT-4o画像は日本語文字を高確率で崩す。タイトルは生成後に Canva 等 or サイトのテンプレ側で重ねる。だからプロンプトで「余白確保・文字なし」を厳守させている。
- **アスペクト比**：アイキャッチ/OG兼用なら横長（1792×1024 ≒ 16:9）。「横長16:9で」と必ず明示。
- **反復のコツ**：1発で決めず、まず②or①で4案出し→気に入った構図を「この構図のまま、フクロウをもう少し小さく」「余白を右上に」等の差分指示で詰める。
- **トーンの肝**：「終活＝暗い」を避け、owl-character.png の“ほんわか”に寄せること。彩度を上げすぎると安っぽくなるので「淡く・上品に」を効かせる。
- **書き出し後**：幅1200pxほどに最適化 → microCMSメディアにアップロード → 記事の eyecatch に設定（本文ではなく eyecatch フィールド）。
