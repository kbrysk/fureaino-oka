# M&A 3,000万円を狙う「生前整理ナビ」戦略統合

買い手が見るのは **「PV数」ではなく「送客の質」と「保有データ（Zero-Party Data）」**。  
本ドキュメントは、事業計画書・フィージビリティ調査に基づく**キーワード戦略**・**DD改善の考え方**・**SEO×PLG**・**ネーミング**を一括で定義する。

## 1. M&A向けキーワード戦略マップ（4フェーズ）

### フェーズ1：集客の入り口（悩み・共感系）

**目的:** 大手未参入の「心理的ハードル」「家族間トラブル」で検索上位を取り、ドメインパワーを育てる。YMYL直撃を避け「生活の知恵」でポジション取る。

| キーワード群 | 検索意図 | 記事→CVの導線 |
|--------------|----------|----------------|
| 実家 片付け 親 説得 / 親 捨てられない 病気 | 片付けない親へのイライラ・共感 | 「親と喧嘩しない魔法の言葉」→ **進捗管理アプリ（親との共有機能）** 登録 |
| 実家 片付け どこから / 生前整理 やり方 リスト | 物量に圧倒され思考停止 | 「場所別ロードマップ」→ **片付け診断シミュ／チェックリスト** でリスト化 |
| 生前整理 疲れた / 実家 片付け 業者 頼みたい | 自力の限界・外注の心理的許可 | 「プロに頼むのは親孝行」→ **不用品回収 一括見積もり** 送客 |
| デジタル遺品 パスワード わからない | 死後スマホ・口座凍結の不安 | 「デジタル資産目録テンプレ」配布 → **会員登録（資産情報取得）** |

### フェーズ2：マネタイズ直結（買取・処分系）

**目的:** 比較的ハードルの低い「不用品回収・買取」で初期収益を作り、サイトの収益性を証明する。

| キーワード群 | 検索意図 | 記事→CVの導線 |
|--------------|----------|----------------|
| 着物 処分 罪悪感 / 古い着物 捨て方 | 思い出品をゴミにする罪悪感 | 「リユース（誰かに着てもらう）」価値転換 → **着物買取** 送客 |
| 仏壇 処分 費用 / 位牌 供養 | 宗教的タブー・バチへの恐怖 | 「お焚き上げ」相場解説 → **遺品整理・供養サービス** 送客 |
| コレクション 処分 夫 / 骨董品 価値 わからない | 家族の趣味品が邪魔・価値不明 | 「鑑定は無料」→ **出張買取** 送客 |

### フェーズ3：高額売却用（不動産・相続系）

**目的:** 売却時のバリュエーションを跳ね上げる「本丸」。直接「不動産売却」は競合強→「維持費」「リスク」からリスト化。

| キーワード群 | 検索意図 | 記事→CVの導線 |
|--------------|----------|----------------|
| 空き家 維持費 シミュレーション / 実家 固定資産税 いくら | 相続後のコスト不安・負動産化の恐怖 | **地域別維持費計算ツール** → 住所・広さ入力で不動産リードとしてタグ付け |
| 実家じまい 費用 / 実家 解体 費用 | 家をたたむ総額・売却意識の芽生え | 解体費用相場 → **不動産一括査定／空き家活用** 送客 |
| 生前整理 いつから | 元気なうちに整理したいがきっかけなし | 年齢別リスク解説 → **エンディングノート／資産入力** 誘導 |

### フェーズ4：指名検索・ツールブランド化

**目的:** M&Aで「独自ツール」「指名検索」はGoogleアップデートのリスクヘッジとして高評価。ツール名を記事内で必ず紹介しブックマーク・SNS拡散を狙う。

| キーワード例 | 戦略 |
|--------------|------|
| 生前整理 アプリ / 実家 片付け リスト エクセル / ゴミ 分別 判定 | 記事内で「無料で使える便利なツール」として自社ツールを紹介。ツール名で検索される状態を作る。 |

## 2. 記事の黄金フォーミュラ（入り口→中間→出口）

1. **入り口（記事）:** 「片付けが大変」「親が頑固」など **共感（Emotional）** から入る。  
2. **中間（論理）:** 「まずは現状を把握しよう」「資産を可視化しよう」と **論理（Logic）** を展開。  
3. **出口（ツール）:** 「この無料ツールで診断できます」「このアプリで親と共有できます」と **行動（Action）** を促し、会員登録させる。

**最優先で着手する記事:**  
競合が弱く深い悩みの **「親の説得（心理系）」** と **「着物・コレクションの罪悪感（買取系）」** を先行投下 → トラフィックと小規模売上 → その実績でツール開発・不動産系キーワード拡大。

## 3. DD指摘と改善の対応（要約）

- **Coming Soon 抹消:** 空き家税金シミュ・査定ツールを簡易版で実装しTOPに配置済み（`DD_IMPROVEMENTS_APPLIED.md` 参照）。  
- **高単価リード前面:** ホームを「危機感＋3分診断」FVに変更。ツールをメイン、チェックリスト等は進捗管理として下段に配置済み。  
- **チェックリスト:** 項目チェック時にアフィCTAモーダル、100%で家族共有アンロック済み。  
- **資産登録:** 売却・処分意向で無料比較モーダル、資産総額バナーでフィードバック済み。  
- **家族招待:** ホーム・サンクス・チェック100%で招待CTAを配置済み。  
- **未対応・今後の拡張:** 金額入力の代わり「住所・写真でAIが金額を出す」体験、診断結果のLINE共有（バイラル）、Programmatic SEO（地域×手続きの動的ページ）。

## 4. 広告ゼロでグロース：「SEO×PLG」

### 診断結果をバイラルさせる（PLG）

- **例:** 「実家じまい力診断」「相続争族度チェック」を作成。  
- 結果画面に「この結果をLINEで家族に送って、会議を開きましょう」ボタン → 1人が親・兄弟を連れてくる構造で **CPAゼロ** を狙う。

### Programmatic SEO

- 記事の手書きだけでなく、**ツールが自動生成するページ**をインデックスさせる。  
- 例: 「世田谷区の粗大ゴミ・遺品整理マップ」「大阪市の空き家補助金シミュ」など **「地域名 × 手続き/計算」** で動的ページを大量生成し、CVに近いニッチKWを総取り。

## 5. ネーミング提案（M&A・SEO・シニア訴求）

買い手が「このドメイン（指名検索）に価値がある」と判断し、かつ **65歳以上のアクティブシニアに刺さる** 名前が望ましい。「生前整理」は「死」を連想させ重いため、**前向き・簡単・家族のため** に変換しつつ **ビッグKW** を含める。

| 案 | 名前例 | 特徴 |
|----|--------|------|
| **SEO・直球** | 実家じまいナビ / **実家スッキリ** | 「実家」で不動産・買取買い手に価値が伝わる。トレンドワードで認知が早い。 |
| **ポジティブ** | あしたの準備 / これなら（Corekara） | 死のイメージを消し登録ハードルを下げる。 |
| **楽さ訴求** | ラク整理 / パシャっと整理 | 「簡単」「無料」と相性が良く、PLG・口コミに効く。 |
| **家族・絆** | 家族ノート / バトン（Baton） | 想い・共有機能への誘導。SEOはタイトルで補足が必要。 |

### 推奨（プロフェッショナル案）

**『実家スッキリナビ』** または **『実家スッキリ』**

- **「実家」** → 不動産・空き家ビジネスへの拡張性（買い手メリット）が明確。  
- **「スッキリ」** → 片付けの快感・ネガティブさの軽減。シニアは「スッキリ」「解決」系に馴染みがある。  
- ロゴは丸みのあるゴシック＋緑（安心）やオレンジ（活発）でアクティブシニアに合わせる。
