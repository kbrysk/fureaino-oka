## タスク形式：合言葉を入力してもらう設定

依頼を**タスク形式**で作成し、ワーカーに「合言葉」を1行で入力してもらうようにします。**必須ワード**を設定すると、合言葉が一致した場合のみ提出が通るため、サイト・フォームを完了した方だけ報酬対象にできます。

### 作業内容（タスク）の設定手順

1. **「作業内容」でタスクを追加する**  
   依頼作成画面の「作業内容 必須」で、左メニューから **「文字入力（一行）」** を選択する。

2. **タイトルを入力する**  
   「フォームのタイトルを入力」欄に、例えば次のどれかを入れる。  
   - `フォーム完了後に表示された「合言葉」をそのまま入力してください`  
   - `合言葉`  
   - `アンケート完了画面に表示されている合言葉を入力してください`

3. **入力必須にする**  
   「入力必須」にチェックを入れる。

4. **文字数制限を設定する（任意）**  
   「文字以下」に、合言葉の文字数より少し多めの数字を入れる（例：50）。  
   合言葉が「【 ふれあいテスト2025 】」なら約15文字なので、50文字以下で十分。

5. **必須ワードを設定する（合言葉の照合）**  
   「詳細を追加」の **「+ 必須ワード」** をクリックし、**Googleフォームのサンクスページに表示している合言葉を一字一句そのまま**登録する。  
   - 例：`【 ふれあいテスト2025 】`（スペースや括弧も含めて同じにする）  
   - フォーム側の `CONFIG.AIKOTOBA` と完全に同じ文字列にすること。

6. **説明文を追加する（任意）**  
   「+ 説明文」で、次のような案内を入れるとワーカーが迷いにくい。  
   - `アンケート送信後の画面に表示される「今回の合言葉はこちらです」の下の文字列を、カギ括弧も含めてコピーし、そのまま貼り付けてください。`

7. **「編集終了」を押す**  
   設定を保存する。

### 運用の流れ

1. ワーカーが応募 → 採用  
2. 依頼者からメッセージで「サイトURL」と「GoogleフォームのURL」を送る  
3. ワーカーがサイトを利用 → フォーム回答 → サンクスページで合言葉を確認  
4. ワーカーがクラウドワークスのタスクで「合言葉」を入力して提出  
5. 必須ワードと一致していれば提出完了。不一致の場合はエラーになり、正しい合言葉の入力が必要

### 注意

- **合言葉はフォームとタスクで完全に一致させる**  
  フォームの `create-user-test-form.gs` の `CONFIG.AIKOTOBA` と、クラウドワークスの「必須ワード」に同じ文字列を入れてください。  
- 合言葉を変更するときは、**フォームのサンクスページ**と**クラウドワークスの必須ワード**の両方を同じ内容に更新してください。
