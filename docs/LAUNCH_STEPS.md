# Phase 0: 準備

## 0.1 事業用 Google アカウントの作成（名義管理）

**M&A・譲渡の観点**: 個人のプライベートなメアドやカードで契約すると、譲渡時に権利関係が曖昧になり、買収監査（DD）で指摘されやすくなります。**最初から事業専用の Google アカウント（Gmail）を作成し、Vercel・Supabase・GA4・Search Console・ASP・ドメイン管理など「すべてそのアカウントで紐付ける」**ようにしてください。

### 手順

1. **事業用 Gmail** を新規作成（例: `seizenseiri-navi@gmail.com` や 独自ドメインの Google Workspace）。
2. そのアカウントで **Vercel・Supabase・Resend・CMS・LINE・X・Instagram** の登録・契約を行う。
3. ドメインの **Whois 管理者** も事業用メール or 代表者メールに統一する。
4. 契約書・請求書の名義を「個人」か「法人」か決め、一貫させる。

- [ ] 契約ツール・SNS・ドメインの登録をすべて事業用アカウントで行う方針を徹底

## 0.2 規約・ポリシードキュメントの作成

**M&A・ASP 審査の観点**: 利用規約・プライバシーポリシーがないと、ASP のメディア審査に落ちるだけでなく、**買収監査（DD）で一発アウト**になります。特に **免責事項（診断結果はあくまで目安）** と **個人情報の取り扱い** は必須です。

### 必要なドキュメント

| ドキュメント | 内容の要点 |
|--------------|-------------|
| **利用規約** | サービスの利用条件、禁止行為、免責（診断結果・シミュレーションは目安であり、専門家の相談に代わるものではない等）、規約変更の手続き。 |
| **プライバシーポリシー** | 収集する個人情報（メール、LINE ID 等）、利用目的、第三者提供（ASP・提携先）、Cookie・アクセス解析（GA4）、問い合わせ先。 |
| **免責事項** | 診断ツール・シミュレーターの結果は「参考情報」であり、法律・税務・不動産の専門家の助言に代わるものではない旨を明記。 |

### 手順

1. テンプレート or 弁護士作成のひな形をベースに、**サイト名・サービス内容・収集データ・ASP 連携** に合わせて編集する。
2. `/terms`（利用規約）、`/privacy`（プライバシーポリシー）などの固定ページを用意し、**フッターからリンク**する。
3. **Phase 1 終了〜公開前** に必ず掲載し、メール登録・診断開始前の同意フローがあれば「プライバシーポリシーに同意する」を入れる。

- [ ] プライバシーポリシーのドラフト作成（個人情報・Cookie・GA4・第三者提供を記載）
- [ ] 免責事項を利用規約 or 各ツールページに明記

# Phase 1: ドメイン取得

## 1.1 新規ドメイン vs 中古ドメイン

|  | 新規 | 中古（オールドドメイン） |
|--|------|--------------------------|
| **メリット** | 履歴がクリーン、即取得しやすい | 既存のドメインオーソリティ・被リンクを引き継げる可能性 |
| **デメリット** | ゼロから評価が積み上がる | ペナルティ・スパム履歴のリスク、選定に手間 |
| **コスト** | 約1,000〜1,500円/年（.com） | 数千円〜数十万円（ランク・年数による） |

**生前整理ナビのように pSEO・記事を軸にする場合**: 中古ドメインで「関連キーワード・ジャンルに近い」「ペナルティなし」のものを選ぶと、初期の評価がつきやすくなる場合があります。ただし **2024年以降、中古ドメインの不正利用は Google のスパムポリシー対象**のため、**正当なコンテンツで運用する前提**で選ぶことが必須です。

## 1.2 中古ドメインの購入手順（概要）

### Step 1: 販売サイトで検索・選定

| サイト | 特徴 | URL 例 |
|--------|------|--------|
| **お名前.com 中古ドメインオークション** | オークション形式。用途別（SEO・アフィリエイト等）で検索可能 | [お名前.com オークション](https://www.onamae.com/campaign/auction/) |
| **中古ドメイン販売屋さん** | 毎週水曜 10:00 新入庫。早い者勝ち。BLD・AGE・備考で絞り込み | 検索で「中古ドメイン販売屋さん」 |
| **オーラン** | ランク（ブロンズ〜ウルトラ）別。品質指標あり | 検索で「オーラン 中古ドメイン」 |

### Step 2: 選定時の確認項目（中古の場合）

- [ ] **ペナルティ履歴**: Google で `site:ドメイン名` や「ドメイン名 ペナルティ」で検索し、過去のスパム運用がないか確認
- [ ] **被リンク品質**: 低品質リンク比率が高くないか（40% 以下が目安とする説あり）
- [ ] **ジャンル・キーワード**: 終活・生前整理・相続・片付け・不動産など**関連ジャンル**のドメインが望ましい
- [ ] **年数（AGE）**: 1年以上あると評価されやすいとする説あり
- [ ] **Whois**: 取得後に管理者情報を自社・個人に変更できるか確認

### Step 3: 購入・支払い

- 各サイトの「カート」「購入」から手続き。**お名前.com** の場合は「ドメイン取得」と同様の流れで、取得後はお名前.com の管理画面で DNS を操作可能。
- **中古ドメイン販売屋さん**など: 支払い後、**プッシュ（移管）** または **お名前.com 等のレジストラへの転送**で受け取り。転送には数日〜1週間かかる場合あり。

### Step 4: 取得後の設定（新規・中古共通）

1. **Whois 情報**: 管理者・連絡先を現在の運用者に更新
2. **ネームサーバー**: 後述の Vercel 用（`ns1.vercel-dns.com` 等）に変更するか、**DNS レコード** で CNAME を Vercel 向けに設定
3. **SSL**: Vercel にドメインを追加すると自動で HTTPS が発行される
4. **Search Console**: 公開後にドメインを登録し、ペナルティやインデックス状況を確認（Phase 4 参照）

### 1.2.2 選ぶときの確認項目（詳細）

#### ペナルティ履歴の確認

| 方法 | 手順 | 目安 |
|------|------|------|
| **site: 検索** | Google で `site:取得予定ドメイン名` を検索する。 | ヒットが **0件** かつ過去にサイトがあった場合は、インデックス削除（ペナルティの可能性）を疑う。新規で何も運用していないドメインなら 0 件でも問題ない場合あり。 |
| **Wayback Machine** | [archive.org](https://archive.org/web/) で `https://取得予定ドメイン名` を検索。 | 過去のページ内容がスパム・アダルト・違法でないか確認。終活・暮らし・情報系など**ジャンルが近い**と安心。 |
| **ブラックリスト** | [aguse.jp](https://aguse.jp/) や Trend Micro Site Safety Center でドメインを検索。 | マルウェア・フィッシング等でブラックリスト入りしていないか確認。 |
| **Search Console（購入後）** | ドメインを取得し、Vercel でサイトを公開したあと、Search Console の「ドメイン」プロパティで追加。 | 「セキュリティと手動対応」に手動ペナルティの通知がないか確認。サイトマップを送信して「送信できた＝ペナルティなし」とする説もある。 |

**注意**: Google は「ドメインのリセットボタンはない」としているため、**手動ペナルティがついたドメインは解除が難しく、別ドメインの取得を検討**した方がよい場合があります。

#### 被リンク品質

- 販売サイトによっては **「低品質リンク比率」** や **「IP分散率」** が表示される。低品質リンクが 40% 以下、IP 分散 60% 以上を目安とする説あり。
- ツール（Ahrefs, SEMrush, 無料なら Ubersuggest 等）でドメインを調べ、**被リンク元がスパムサイト・リンクファームでないか**をサンプル確認すると安心。

#### ジャンル（終活・生前整理など）

- **Wayback Machine** で過去のタイトル・見出しを確認し、「終活」「生前整理」「相続」「片付け」「暮らし」「情報」など**現在のサイトと近いジャンル**だと、評価が引き継がれやすく、Google からも「テーマの急変」と見られにくい。
- 販売サイトの「備考」で **「特」（特化系）「地」（地域）「教」（教育・情報）** などがついていると、関連ジャンルで使われていた可能性が高い。

#### 年数（AGE）

- **ドメインの登録年数** や **最初にインデックスされたとみなされる年数** が長いほど、検索エンジンで「枯れたドメイン」として扱われやすいとする説あり。1年以上、できれば 2〜3 年以上あるとよいとする目安が多い。

#### Whois（取得後に更新できるか）

- 購入前に、そのドメインが **ロック解除・移管可能** か、**Whois 公開代行を外して管理者情報を自分に変更できるか** を販売サイトの説明で確認する。お名前.com で受け取る場合は、管理画面から WhoIs 情報の変更が可能。

### 1.2.3 購入後の設定（詳細）

#### Step A: Whois 情報の更新

1. ドメイン管理画面（お名前.com の「ドメイン一覧」→ 対象ドメイン → 「Whois 情報」）を開く。
2. **管理者（Registrant）・連絡先** を自分（または運営法人）の名前・メール・住所・電話番号に変更。
3. **Whois 公開代行** を使っている場合は、必要に応じて「公開」に切り替え（Search Console の所有権確認でメールを受け取る場合など）。
4. 保存後、最大 24〜48 時間で公開 Whois に反映される場合がある。

#### Step B: ネームサーバーを Vercel 向けに変更

1. **Vercel** の Project → **Settings → Domains** で「Add」→ 取得したドメイン（例: `seizenseiri-navi.com`）を入力。
2. Vercel が **推奨する設定** を表示する:
   - **ネームサーバー方式（推奨）**: `ns1.vercel-dns.com`, `ns2.vercel-dns.com` の2つを指定。
3. **お名前.com** の場合:
   - 「ドメイン一覧」→ 対象ドメイン → 「ネームサーバーの設定」を開く。
   - 「その他のネームサーバーを利用」を選択し、上記 2 つを入力して保存。
4. **反映待ち**: 数分〜最大 48 時間。Vercel の Domains 画面で「Verified」になれば完了。
5. **ルートドメイン（apex）と www**: Vercel は両方に対応。`example.com` と `www.example.com` のどちらも Add して検証する。

**CNAME 方式**（ネームサーバーを変えたくない場合）:
- ルートドメインでは CNAME が使えないレジストラが多いため、**www のみ**（例: `www.seizenseiri-navi.com`）を CNAME で `cname.vercel-dns.com` に振る運用になる。ルートを www にリダイレクトする設定は Vercel 側で可能。

#### Step C: SSL（HTTPS）

- Vercel にドメインを追加し、ネームサーバー（または CNAME）が正しく向いていれば、**Let's Encrypt による SSL 証明書が自動発行**される。手動でインストールする作業は不要。
- 数分〜数十分で「Secure」になり、`https://取得ドメイン` でアクセスできる。

#### Step D: Search Console（公開後）

- 本番公開後、Phase 4 のとおり **Google Search Console** にドメインプロパティで追加し、**サイトマップ**（`https://取得ドメイン/sitemap.xml`）を送信。
- 「ページ」「カバレッジ」「セキュリティと手動対応」で、ペナルティやインデックスエラーがないか確認する。

---

### 1.2.4 注意：中古ドメインの不正利用と Google のスパムポリシー

- **Google の公式方針**: 期限切れドメイン（中古ドメイン）を取得し、**内容を無関係なテーマに急に変えて低品質・スパム的なコンテンツで埋める**ような利用は、**スパムポリシー違反**の対象となり得る（2024年3月のコアアップデート以降、明文化されている）。
- **「正当なコンテンツで運用する」とは**:
  - 中古ドメインであっても、**終活・生前整理・相続・片付けなど、ドメインの過去のジャンルや現在のサイトテーマと整合した、役立つオリジナルコンテンツ**で運用する。
  - 被リンクを「買う」・リンクファームに参加する・キーワード詰め込み・クローク等は行わない。
- こうした前提で選び・運用すれば、中古ドメインの**ドメインオーソリティや経年**を活かしつつ、リスクを抑えられます。

# Phase 2: 契約ツール・LINE・SNS アカウント

## 2.1 契約ツールの順序とチェックリスト

| # | ツール | やること | リンク・メモ |
|---|--------|----------|--------------|
| 1 | **Vercel** | アカウント作成 → **Pro** にアップグレード（商用利用のため必須） | [vercel.com](https://vercel.com) |

## 4.2 GA4 イベント・コンバージョン設定（マイクロコンバージョン計測）

**事業グロースの観点**: GA4 を入れるだけでは「どこを改善すればいいか」がわかりません。**診断完了・LINE ボタンクリック・アフィリエイトリンククリック** を **イベント（コンバージョン）** として計測し、CVR 改善の根拠にします。

### 計測したいイベント（例）

| イベント名（例） | 内容 | 用途 |
|------------------|------|------|
| `diagnosis_complete` | 診断完了（実家じまい力・空き家リスク・相続シミュ等の「結果表示」） | 診断 CVR の把握 |
| `line_button_click` | LINE で結果を受け取るボタンのクリック | LINE 導線の効き具合 |
| `affiliate_click` | アフィリエイトリンク・CTA のクリック（`/guide` や ASP リンク） | 収益導線の効き具合 |
| `lead_submit` | メール登録・ガイドブック申込の送信完了 | リード獲得 CVR |

### 実装方法（GTM 推奨）

| 方法 | 内容 |
|------|------|
| **Google タグマネージャ（GTM）** | GTM コンテナをサイトに 1 つだけ埋め込み、GTM 上で「トリガー（クリック・ページビュー）」「タグ（GA4 イベント）」を設定。コードを触らずにイベント追加・変更が可能。**推奨**。 |
| **コードで gtag 送信** | 該当ボタンやページで `gtag('event', 'diagnosis_complete', { ... })` を呼ぶ。Next.js なら `useEffect` やクリックハンドラで送信。 |

### 手順（GTM の場合）

1. **GTM** [tagmanager.google.com](https://tagmanager.google.com) でコンテナを作成（ウェブ）。
2. GTM の「タグ」で **GA4 イベント** を追加。トリガーを「クリック - リンクの URL に guide が含まれる」等で設定。
3. 診断完了は「ページビュー - 結果ページの URL」や「カスタムイベント」でトリガーを組む。
4. GA4 の **管理 → イベント** で、上記イベントを **「コンバージョン」としてマーク** する。
5. 本番で GA4 の「リアルタイム → イベント」に、クリックや診断完了が飛んでいるか確認する。

### チェックリスト

# Phase 6: リンク管理（リンク切れ防止・チェック体制）

## 6.1 リンク切れの防止策

| 対策 | 内容 |
|------|------|
| **相対パス** | サイト内リンクは `/articles/xxx` のように相対パスで記述し、ドメイン変更時も切れにくくする |
| **アフィリエイト URL** | ASP の「リンクツール」で発行された URL をそのまま使う。変更時は ASP 側で確認 |
| **外部リンク** | 行政サイト・公式窓口などは URL が変わりやすいため、定期的なチェック対象にする |
| **リダイレクト** | 記事の URL を変更した場合は、旧 URL → 新 URL の 301 リダイレクトを設定（Next.js の redirects） |

## 6.4 トラブル対応フローの策定

**運用責任者の観点**: 「サイトが落ちた」「フォームからスパムが大量に来た」「LINE が BAN された」等のトラブル時に、**誰が何を確認し、どう対処するか** を決めておかないと、自分が倒れたらサイトが止まります。

### 想定トラブルと対応の目安

| トラブル | 確認すること | 対応の目安 |
|----------|--------------|------------|
| **サイトが落ちた・表示されない** | Vercel のダッシュボードでデプロイ・エラー有無を確認。ドメインの DNS が切れていないか確認。 | ロールバック、環境変数の見直し、Vercel サポート or ドメイン業者に問い合わせ。 |
| **フォームからスパムが大量に来た** | Supabase の `leads` テーブルや Resend の送信ログを確認。reCAPTCHA や honeypot の導入を検討。 | 一時的にフォームを無効化 or CAPTCHA 強化。IP ブロック（Vercel / Edge で対応可能なら検討）。 |
| **LINE が BAN された・使えなくなった** | LINE 公式アカウントの通知・規約違反の有無を確認。 | LINE サポートに問い合わせ。規約に沿った運用に修正。診断結果送信を「メールでも受け取る」等の代替導線を用意。 |
| **ドメインの SSL が切れた** | Vercel の Domains で証明書の状態を確認。 | Vercel 側で再発行 or ネームサーバー・CNAME の設定ミスを修正。 |
| **GA4 や Search Console にアクセスできない** | 事業用 Google アカウントの権限・2 段階認証を確認。 | アカウント復旧 or 別アカウントにプロパティを移行。 |

### フロー策定の手順

1. 上記の「想定トラブル」を一覧にし、**一次対応者（自分 or 外注）** と **確認手順・エスカレーション先** を書く。
2. ドキュメント（Notion・Google ドキュメント・リポジトリの `docs/INCIDENT.md` 等）にまとめ、共有できる状態にする。
3. 定期的に「サイトが落ちた想定」で手順を読み返し、不足があれば追記する。

| **SNS でのティザー・告知** | Phase 2 で開設した X・Instagram で、「本日〇〇を公開しました。無料の診断ツール・ガイドブックがあります」と投稿。ハッシュタグ（#生前整理 #終活 等）を付ける。 |
| **バズ施策（任意）** | 診断結果の「シェア」ボタンや「X でつぶやく」リンクから拡散されやすい文言・画像を用意する。インフルエンサーやメディアへのプレスリリース送付も検討。 |

### チェックリスト

- [ ] 公開日を決め、その前に SNS でティザー投稿（「もうすぐ公開」等）を 1〜2 回
- [ ] 公開当日に SNS で告知投稿
- [ ] 友知人にシェア依頼（リスト化して漏れなく）
- [ ] 1〜2 週間後、GA4 のセッション数・コンバージョン数・Supabase のリード数を確認
