# GA4 ボット除外 ＆ クリーン数字 完全ガイド

> 4専門家パネルの合意：DDの不意打ち（Direct43%・ボット19%）を消すことは破談リスク（価格毀損 数百万）の低減に直結。
> **重要な事実：GA4の管理UIフィルタでは、ヘッドレス・データセンターボット（IP動的）を完全には除外できない（Google自身の限界）。**
> よって本ガイドは「①UIフィルタ＝出来る範囲のお掃除」＋「②クリーン数字の再現可能な算出＝DD防御の本丸」の2層で構成する。

---

## 層① GA4管理UIフィルタ（あなた・5分・ongoing hygiene）

GA4 → 管理（左下歯車）→ プロパティ列「データの設定 → データフィルタ」

### 1-1. 内部トラフィックの除外（自分の閲覧を除外）
1. 管理 → データストリーム → 該当ストリーム → タグ設定を行う → 内部トラフィックの定義
2. 「ルールを作成」→ ルール名: internal、traffic_type 値: internal、IPアドレス: あなたのグローバルIP（"自分 IP" で検索して確認）を `IP アドレスが次と等しい` で追加
3. 管理 → データフィルタ → 「Internal Traffic」フィルタを **有効** に（既定はテスト中）

### 1-2. 既知ボットの除外（既定でON・確認のみ）
- GA4は IAB既知ボットを自動除外する設定が常時ON（設定項目なし）。ただし**データセンターのヘッドレスbotはここで捕捉されない**（だから層②が必要）。

### 1-3. （任意・限定効果）デベロッパートラフィック
- 開発・プレビュー環境からの流入は debug_mode で分離可能だが、本件のボットには無効。

> **UIフィルタの限界**：本件の主犯（/area・/cost に0秒で着地するDC合成流入）はIPが動的で参照元も無いため、GA4 UIでは狙い撃ちできない。UIフィルタは「自分の閲覧除外」程度の効果と割り切る。

## 層② クリーン数字の再現可能な算出（DD防御の本丸・Claude実装済み）

GA4の「ランディングページ」レポートをCSVエクスポートし、ボット署名をコードで除外する。**透明・再現可能・買い手に算出ロジックごと開示できる**のが強み。

### 実行
```
node scripts/ga4-clean-numbers.mjs "<GA4ランディングページCSVのパス>"
```

### ボット署名（除外ルール・明示）
- エンゲージ0秒 かつ ランディングが `/area`・`/cost` 完全一致（合成流入の指紋）
- `.html` 終端の旧静的パス（中古ドメイン残骸クロール）
- `(not set)` かつ エンゲージ<5秒
