# Programmatic SEO：地域 × 手続き データスキーマ

「地域名 × 手続き」で動的SEOページを量産するためのデータ構造です。

## CSV カラム

| カラム名 | 必須 | 説明 | 例 |
|----------|------|------|-----|
| 都道府県 | ○ | 都道府県名 | 東京都 |
| 市区町村 | ○ | 市区町村名（区・市・町・村） | 世田谷区 |
| 粗大ゴミ申し込みURL | ○ | 粗大ゴミの申込・案内ページ | https://www.city.setagaya.lg.jp/... |
| クリーンセンター電話番号 | ○ | 清掃事務所・クリーンセンターの問い合わせ先 | 03-3700-xxxx |
| 遺品整理業者備考 | ○ | 遺品整理の案内文 | 提携業者に無料相談 |
| 補助金備考 | - | 空き家解体・補助金・3000万円控除の案内（省略時は共通文） | 世田谷区では… |
| 片付け相場備考 | - | 遺品整理・片付け相場の案内（省略時は共通文） | 2LDKで20〜40万円程度の目安… |

## 全国展開

- **データ生成**: `content/area-data/municipalities-nationwide.json` に都道府県別の市区町村リストを用意し、`node scripts/generate-areas-csv.mjs` で `areas.csv` を再生成する。既存行（東京23区など実URL・電話あり）は維持され、新規行は粗大ゴミURL＝検索リンク・電話＝要確認で出力される。
- **プレースホルダー**: URLが検索リンクの地域は「〇〇県〇〇市の粗大ゴミ案内を検索」と表示。電話が「要確認」の地域は「お住まいの自治体HP・窓口でご確認ください」と表示。

## 運用

- **MVP**: 東京都23区は実データで運用。全国はプレースホルダーで展開し、順次自治体HPのURL・電話を差し替え可能。
- **更新**: 自治体URL・電話番号は変更があるため、年1回程度の見直しを推奨。
- **遺品整理業者**: 提携アフィリエイトのリストを保持。未提携の場合は「無料一括見積もり」等の汎用CTAへ誘導。
- **補助金・相場**: 市区町村ごとに差をつけたい場合のみ列を埋める。空欄なら共通テキストで表示。

## ファイル配置

- マスタデータ: `content/area-data/areas.csv`
- 動的ルート:
  - `app/area/[prefecture]/[city]/page.tsx` … 粗大ゴミ・遺品整理メイン
  - `app/area/[prefecture]/[city]/subsidy/page.tsx` … 空き家・補助金・3000万円控除
  - `app/area/[prefecture]/[city]/cleanup/page.tsx` … 遺品整理・片付け相場

## URL例

- `/area/東京都/世田谷区` → 世田谷区の粗大ゴミ・遺品整理
- `/area/東京都/世田谷区/subsidy` → 世田谷区の空き家 解体 補助金・3000万円控除
- `/area/東京都/世田谷区/cleanup` → 世田谷区の遺品整理・片付け相場
- `/area/大阪府/大阪市` → 大阪市の粗大ゴミ・遺品整理マップ
