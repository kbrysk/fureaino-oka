# Google AdSense 審査・サイト確認が通らないときのチェックリスト

## 1. コードの配置（実施済み）

- ルートレイアウト `app/layout.tsx` の `<head>` 内に、AdSense のスクリプトを**初期 HTML に含む**形で配置しています。
- パブリッシャーID: `ca-pub-8324936850324481`
- `next/script` の `afterInteractive` ではなく、通常の `<script async src="...">` にしているため、Google のクローラーが取得する HTML にコードが含まれます。

## 2. サイトがクローラーから見えるか

- **SSL エラーを解消する**  
  `www.fureaino-oka.com` で `NET::ERR_CERT_COMMON_NAME_INVALID` が出ている場合、その URL では審査が進みません。  
  → Vercel の Domains で `www` を正しく追加し、証明書を「Valid」にしてください。  
  → 審査依頼は **https で正常に開けるドメイン**（例: `https://fureaino-oka.com`）で行うとよいです。

- **公開されているか**  
  認証や Basic 認証で守られていないか確認してください。審査時はトップページが誰でも閲覧できる状態にします。

- **robots.txt**  
  当プロジェクトでは `/` は許可し、`/api/` のみ禁止しています。トップや主要ページがブロックされていなければ問題ありません。

## 3. AdSense 側の操作

- **「確認」前にデプロイ**  
  コードを入れたら、必ず Vercel などにデプロイし、本番の URL でスクリプトが読み込まれていることを確認してから、AdSense の「確認」を押してください。

- **別の確認方法を試す**  
  「AdSense のコード」で確認できない場合は、AdSense の「サイトの所有権の確認」で **「HTML タグ」（meta タグ）** の方法が表示されていれば、そちらを選んで指定の meta を `app/layout.tsx` の `metadata` の `other` に追加する方法もあります。

- **審査のタイミング**  
  確認が成功したあと、**審査（サイトの審査）** には数日〜数週間かかることがあります。審査中は「審査をリクエスト」を押したあと、結果を待つ形になります。

## 4. 確認用：本番 HTML にコードが出ているか

1. ブラウザで `https://（本番ドメイン）/` を開く（SSL エラーが出ない URL で）。
2. 右クリック → 「ページのソースを表示」。
3. 検索（Ctrl+F）で `adsbygoogle` または `ca-pub-8324936850324481` を検索。
4. `<head>` 内に該当の script タグがあれば、審査用コードの配置は問題ありません。

## 5. まとめ

- コードは **head 内・初期 HTML** に出すように済んでいます。
- **https でエラーなく開けるドメイン**で、**デプロイ済みの状態**で「確認」を実行する。
- 確認が何度も失敗する場合は、**HTML タグ（meta）での確認**に切り替えるか、Vercel のドメイン／SSL 設定を見直す。
