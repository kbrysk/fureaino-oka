# ページ棚卸し診断 ＆ HCS負債マップ（2026-06）

> 読み取り専用診断。データ源：GSC（pages/daily/queries・2026-03-20〜05-29）＋ municipalities.json（pSEO実態）。
> 目的：被リンク施策の前に「どのページが生きていて、どれがHCS負債か」を確定する。

## エグゼクティブ・サマリー（冷徹版）

1. **pSEOエリアページ 5,225枚のうち、84%（約4,380枚）が直近28日で表示0＝完全な死蔵。**
2. **その死蔵ページは1枚あたり実データ100〜270文字。** 粗大ごみページに至っては中央値**101文字（URL＋電話番号のみ）**。これはGoogleが2024-2025で狙い撃ちしている「scaled content abuse（大量生成の薄いページ）」の教科書的プロファイル。
3. **表示は3月の約250/日 → 5月の約60/日へ75%減。一方で平均順位は70位→40位台に改善。** これは「Googleが薄い長尾ページをSERPから間引いている」典型シグナル。**アルゴリズムは既に減点を始めている**（手動対策ではないが実質的なHCS抑制）。
4. **100本超のコンテンツ記事は検索的にほぼ死亡。** `/articles/` 系の合計表示は約16回・クリック0。量産は検索可視性をほぼ生んでいない。
5. **唯一の勝ち筋＝tax-simulator（平均17位＝2ページ目）。** コンテンツではなく「ツール」が最良順位。
6. **粗大ごみページは自社ルール違反でもある**（`自治体の電話番号・tel露出禁止`なのに電話番号を露出）。

→ **結論：被リンクを積む前に、この4,380枚の薄いpSEOを必ず処理する。** 沈んだ船に荷を積んでも沈むだけ。これは攻め（HCS減点解除）と守り（DDで買い手が真っ先に減点する箇所の除去）の両方。

## 5. 診断からの処方（着手順）

### Step 0：ライブGSC再取得（要再認証・1回）
6〜12ヶ月推移／country別／インデックス カバレッジを確定。
```
gcloud auth application-default login --scopes=openid,https://www.googleapis.com/auth/cloud-platform,https://www.googleapis.com/auth/analytics.readonly,https://www.googleapis.com/auth/webmasters.readonly
gcloud auth application-default set-quota-project seizenseiri-497212
npm run gsc:report
```

### Step 1：HCS負債の清算（最優先・攻守両用）
- **粗大ごみpSEO 1,726枚**：(a)noindex化、または(b)県単位の1ページに統合（「広島県の粗大ごみ・不用品の出し方」等の厚いハブ1枚＋市区は内部アンカー）。電話番号露出も同時に除去。
- **表示0の補助金pSEO**：データが薄い市区はnoindex、補助金が実在し情報が厚い市区のみindex維持。
- 効果：HCS減点解除／クロールバジェットを生きたページへ／DD減点箇所の除去。

### Step 2：生きてる地域クラスタを押し込む
広島・千葉・岡山・佐賀の表示獲得ページに内部リンク集約＋オンページ最適化＋少数の権威リンク。**最初の本物のクリック＝査定CVを取りにいく。**

### Step 3：被リンクは「補助金DB」一点突破（協会メールは廃止）
全国補助金DBを引用可能な一次データ資産に磨き、不動産・相続・報道メディアにデータPR。

### Step 4：tax-simulatorを伸ばす
唯一2ページ目の資産。被リンク・内部リンク・機能拡張で1ページ目へ。査定CVの主動線にする。
