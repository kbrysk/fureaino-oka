# 海外トラフィック調査レポート（2026-06）

> 調査対象：GA4「ユーザー属性の詳細（市区町村／地域）」2026-03-01〜06-10 ＋ GSC検索クエリ（2026-06-01取得分）
> 問い：「なんで海外の人がこのサイトに来てるのか。どういうクエリで来ているのか」

## 結論（先に3行）

1. **海外の人は『検索クエリ』では来ていない。** GSCの実クエリは**1,000件すべて日本語**で、外国語クエリは**0件**。サイトは海外の検索結果に一切出ていない。
2. **GAの「海外ユーザー」の正体は大半が米国データセンターのボット／クローラー。** 地域ベースで約19%、都市ベースでは既知データセンターだけで514人。多くがエンゲージ時間ほぼ0秒。
3. **本物の海外在住ユーザーはごく少数（数人規模）。** 日本語が読める海外在住日本人が、日本語の終活・実家じまいワードで来ているだけ。

## 4. なぜこれが重要か（EXIT＝M&A視点）

- **ボット流入はGAの数字を水増しする。** M&A査定でGAの「ユーザー数」をそのまま使うと、実態より良く見える一方、買い手のDDで「海外ボット2割」が露見すると逆に信頼を損なう。
- **判断はGA総数でなく「①GSCの日本語検索 ②エンゲージのあったセッション」で見るべき。**
- 海外向けSEO（O1海外クラスタ）は**被リンク獲得・E-E-A-T向上が目的**であり、海外からの集客が目的ではない。今回のデータはその方針と矛盾しない。

## 5. 推奨アクション

1. **GA4のbotフィルタリングを有効化**（管理→データストリーム→「既知のbotを除外」。多くは既定で除外されるが、ヘッドレスbotはGAを発火するため別途IP/地域フィルタが有効）。
2. **データセンターIP/地域をGA4の「フィルタ」または「比較セグメント」で除外**し、実ユーザーだけのダッシュボードを別に持つ（DD提出用の「クリーンな数字」）。
3. **国別の検索実態を確定したい場合**は、GSCを `query × country` 次元で取得する（下記参照）。今回はADC認証切れで取得できなかった。

### GSC国別クエリを取りたい場合（要1回の再認証）

```bash
gcloud auth application-default login --scopes=openid,https://www.googleapis.com/auth/cloud-platform,https://www.googleapis.com/auth/analytics.readonly,https://www.googleapis.com/auth/webmasters.readonly
gcloud auth application-default set-quota-project seizenseiri-497212
npm run gsc:report   # query/page/country/device を logs/gsc/ に出力
```
※ ただしGSCは「Google検索経由」のみ集計するため、ボット流入は元々含まれない。国別で見ても海外クエリが極小なことの裏付けになる。
