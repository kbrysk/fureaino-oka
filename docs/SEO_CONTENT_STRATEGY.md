# 生前整理ナビ：SEOコンテンツ戦略と2年Exitロードマップ

リード獲得型オウンドメディアとして、**アクセス → リード → アフィリエイト/送客 → 数千万円規模でのExit（2年）** を実現するためのSEOコンテンツ設計と、Cursor（AI）を活用した効率的な量産方法。

---

## 1. 2年後の数値目標（逆算）

| 指標 | Year 1 末 | Year 2 末 | 備考 |
|------|-----------|-----------|------|
| **記事数** | 150〜200本 | 350〜450本 | お悩み系ロングテール中心 |
| **月間PV** | 5万〜10万 | 15万〜25万 | オーガニック想定 |
| **月間リード** | 200〜500 | 800〜1,500 | 見積もり・ツール登録・査定申込 |
| **月間送客収益** | 30万〜80万 | 150万〜300万 | CPA・紹介料 |
| **Exit想定** | — | **3,000万〜1億円** | 売上/利益マルチプル＋戦略価値 |

Exitの根拠：  
「ツール会員リスト（Zero-Party Data）」＋「高単価送客実績」が買い手（不動産・リユース・終活企業）のM&Aニーズと合致する前提（事業プランPDFと同様）。

## 2. キーワード・コンテンツ量の目安

### キーワード数

| フェーズ | キーワード数 | 内容 |
|----------|--------------|------|
| **Phase 1（0〜6ヶ月）** | 80〜120 | お悩み系・ずらし（親 説得 片付け、実家 片付け どこから、着物 処分 罪悪感 など） |
| **Phase 2（6〜12ヶ月）** | +80〜120 | 処分・供養・兄弟トラブル・デジタル遺品など拡張 |

**合計：260〜390キーワード**をリスト化し、1キーワード＝1記事を基本とする（一部は1記事で2〜3KWカバー可）。

### コンテンツ量（年間）

- **Year 1**：月12〜18本 → **150〜200本/年**
- **Year 2**：月15〜20本 → **180〜240本/年**（既存記事の更新も並行）

効率化のため、**テンプレート化（構成・見出し型）** と **Cursorでの下書き生成 → 人間の fact-check・トーン調整** が必須。

## 3. アクセス・リード・収益の逆算（前提数字）

### 想定CVR・単価（事業プランPDFベース）

| 導線 | CVR（記事→アクション） | 単価（CPA/紹介料） |
|------|-------------------------|---------------------|
| 不用品・生前整理 見積もり | 0.5〜1.5% | 3,000〜5,000円/件 |
| 出張買取（着物等） | 0.3〜1% | 5,000〜15,000円/件 |
| 不動産一括査定 | 0.1〜0.5% | 1万〜2万円/件 |
| ツール会員登録 | 1〜3% | LTV（後続送客）で評価 |

### 粗計算（Year 2 末イメージ）

- 月間PV **20万** × 平均CTR 2% → ツール/申込ページ **4,000クリック**
- そのうち 20% が何らかのリード化 → **月800リード**
- 不用品 400件×4,000円 + 買取 200件×8,000円 + 査定 50件×1.5万 ≈ **200万〜300万/月**

このレベルが示せれば、**「月商200万〜300万のメディア＋ツール＋リスト」** として数千万円クラスの買い手評価が現実的。

## 4. Cursor（AI）でSEO記事を量産する効率的な使い方

### 基本の流れ

1. **法律・SEOガイド** `docs/CONTENT_LEGAL_AND_SEO_GUIDE.md` を前提に（非弁・非税理士禁止、YMYL、E-E-A-T、押し買い排除）  
2. **キーワード表**（CSV/Notion）で「メインKW・検索意図・見出し案」を管理  
3. **1記事＝1ブリーフ**（タイトル・メタ説明・H2/H3案・E-E-A-T・出典・内部リンク先）を `SEO_ARTICLE_BRIEF_TEMPLATE.md` で用意  
4. **Cursor にブリーフ＋テンプレを渡して下書き生成**  
5. **人間が確認**：法務・医療は専門家チェック、公的出典の有無、トーンは「共感・寄り添い」に調整  
6. **公開**：メタタグ・OGP・内部リンク・CTA（ツール/見積もり）を挿入  

### Cursor で依頼するときの効率化

- **プロジェクトに `docs/SEO_*` を置く**  
  - キーワード一覧、記事テンプレ、ブリーフ例を置いて「この形式で書いて」と参照させる  
- **ルールで固定する**  
  - 「記事は H2/H3 を必ず守る」「リード文150字」「結論を先に」「専門用語は初出で説明」など  
- **1回のチャットでやること**  
  - 例：「`docs/SEO_ARTICLE_BRIEF_TEMPLATE.md` のブリーフに沿って、`/app/articles/[slug]/page.tsx` 用の本文を書いて。キーワードは ○○、文字数2,500字」  
- **バッチ運用**  
  - 同じカテゴリの記事を5本分ブリーフだけ先に作り、Cursor に5本分まとめて「順に下書きして」と依頼 → あとで一括で推敲  

### 品質を落とさないポイント

- **法律・SEOガイド準拠**: `docs/CONTENT_LEGAL_AND_SEO_GUIDE.md` に従う。非弁・非税理士禁止、押し買い排除、YMYL・E-E-A-T の具体策を毎記事で反映。  
- **YMYL に踏み込む部分**（相続・税金・医療）は短文に抑え、公的出典を引用し「専門家に相談を」と明記。  
- **共感トーン**：「〜すべき」より「〜で困っている方へ」「まずは一歩から」。デス・クリーニングの考え方（毎日少しずつ・他者への思いやり）を反映。  
- **内部リンク**：毎記事で「チェックリスト」「はじめかた」「関連記事2〜3本」を必ず入れる。

## 5. 記事の設計（カテゴリ・導線）

### カテゴリ例（ナビ・サイト構造と対応）

| カテゴリ | 例キーワード | 記事のゴール |
|----------|--------------|--------------|
| 親・家族 | 親 説得 片付け、実家じまい 兄弟 トラブル | 共感 → ツール「共有機能」・弁護士コラム |
| 進め方 | 実家 片付け どこから、生前整理 疲れた | ロードマップ → チェックリスト・シミュレーター |
| 処分・供養 | 着物 処分 罪悪感、仏壇 捨てる バチ | 価値転換・供養 → 買取・お焚き上げ送客 |
| 心とからだ | 親 捨てない 病気、溜め込み | 制度・専門家 → 業者紹介 |
| デジタル | デジタル遺品 パスワード わからない | 啓蒙 → ツール登録・目録機能 |
| 資産・家 | 空き家 維持費 シミュレーション、写真 整理 生前 | シミュレーター → 不動産査定・デジタル化 |

### 全記事で共通させる導線

- **リード文のあと**：「まずは現状を可視化」→ チェックリスト／ツール  
- **記事の最後**：CTA「無料で進捗管理」「見積もりはこちら」「査定の目安を見る」  
- **内部リンク**：同じカテゴリ＋「資産・持ち物」「エンディングノート」へのリンクを2〜3本

## 7. Exit につなげるために揃える数字（売上部門・M&A用）

買い手がDDで見るポイント（事業プランPDFと同様）：

1. **送客数と質**：月間の見積もり・査定・買取申込数、成約率の推移  
2. **ツール会員**：登録数、アクティブ率、紐づく属性（年齢・持ち家・進捗）  
3. **トラフィックの質**：オーガニック比率、直帰率、PVあたりリード数  

**「記事数・PV」だけでなく「1PVあたりのリード単価」「ツール経由の高単価リード数」** をダッシュボードで見える化し、**「メディア＋ツール＋リスト」** としての価値を数値で説明できるようにする。
