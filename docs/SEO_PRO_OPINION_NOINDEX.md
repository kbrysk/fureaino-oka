# 100記事を即noindexすべきか — SEOプロフェッショナル意見書

最終更新: 2026-06-01 / 対象: 週末に一括投入した約100記事 / 想定読者: 経営判断者（事業売却まで12ヶ月）

---

## 結論（先に書く）

**「即noindex」は不要。** ただし無条件の放置でもない。「品質スコアリング → 統合・リライト → 8週間観察 → それでも0impのみnoindex」という**4段階の選別プロセス**を回すこと。これが最短かつ最も評価額を毀損しないルートである。

理由を一行で言えば、**いま100記事をまとめてnoindexにする行為は、Googleに対して「このサイトは品質判断ができないので大量に消した」と自己申告するに等しく、ドメイン全体の評価を下げる**からである。事業売却を控えたサイトでこの判断は致命的になりうる。

---

## 1. なぜ「即noindex」がリスクなのか（4つの実害）

### 1-1. インデックス未完了のページにnoindexを入れる無意味さ
GSC の `articles/*` 配下が `impressions=11 / clicks=0`（過去90日）という数字は、**「コンテンツが悪い」のではなく「まだクロール・インデックスが浸透していない」**ことを意味する。週末に100記事を一気に投入したばかりなら、Googleがクロール → インデックス → 評価 → ランキング反映までに**通常3〜6ヶ月**かかる。この時点で「インプレッションが出ない=品質が低い」と判定するのは、種を蒔いた翌日に発芽していないと文句を言うのと同じである。

### 1-2. 「大量に消えるサイト」シグナルでドメイン信用が下がる
Googleは短期間（数週間〜数ヶ月）に大量のページがnoindex化／削除されるサイトを、**サイト品質シグナル（site-wide quality signal）の急変**として記録する。Search Quality Rater Guidelines が言うところの「コンテンツの不安定性」は、サイト全体の rankings に影響しうる。

特に問題なのは、**Helpful Content System（HCS）はサイト単位で評価する**点。100記事を一斉にnoindexすれば「このドメインは AI 生成記事を量産して即座に隠した」という履歴をHCSに刻むことになり、残った優良ページ（`tax-simulator` CTR 3.32% など）まで巻き添えで評価を落とすリスクがある。

### 1-3. 公開直後の記事は3〜6ヶ月の評価期間が必要
SEO業界で言う「Google Sandbox」「Honeymoon Period」など呼称は様々だが、**新規ドメイン／新規ページが安定したランキングに落ち着くまで最低90日**かかるのは経験則として広く共有されている。投入から数日〜数週間で impressions が0でも、それは「評価中」のシグナルでしかない。

### 1-4. 内部リンク網が穴だらけになる
100記事の中には、ピラー記事や既存ツール（空き家リスク診断、相続シミュレーター）への内部リンクのハブとして機能しているページが含まれているはずだ。これらを一斉にnoindexすれば、**`tax-simulator` のような数少ない CTR 3%超のページへの内部リンク評価（PageRank分配）も消失**する。残した優良ページの順位を、自分の手で下げる行為になる。

### 1-5. （追加）改善努力を放棄したシグナル
これはGoogleアルゴリズム上の話ではなく、**M&A時のデューデリジェンス**で買い手が必ず見る指標。「過去半年でインデックス数が30%減」「公開記事の70%をnoindex化」といった履歴は、買い手の評価額算定で必ずディスカウント要因になる。3,000万円の売却額が2,000万円に下振れする説明責任を負える判断か、自問してから動くべきである。

## 2. 正しい対応プロセス（5フェーズ）

### Phase 1（即時／〜3日）: 全記事の品質スコアリング
**`scripts/content/quality-score-articles.mjs` を実行**し、各記事を以下8軸で0-100点採点する：

| 軸 | 配点 | 評価ロジック |
|---|---|---|
| 文字数 | 20点 | 2000字以上=20 / 1500-1999=15 / 1000-1499=10 / それ未満=0 |
| H2見出し数 | 15点 | 5本以上=15 / 3-4本=10 / 1-2本=5 |
| 内部リンク | 15点 | 5本以上=15 / 3-4本=10 / 1-2本=5 |
| 外部出典 | 10点 | 2本以上=10 / 1本=5 |
| 画像 | 10点 | 2点以上=10 / 1点=5 |
| 監修者 | 10点 | あり=10 |
| タグ | 10点 | 3つ以上=10 / 1-2=5 |
| description | 10点 | あり=10 |

判定: **60点以上=健全 / 40-59点=要改善 / 40点未満=重大な懸念**

CSV/JSON で `logs/content-quality/quality-report-YYYY-MM-DD.csv` に出力されるので、Excel で並べ替えて重大懸念ページを特定する。

### Phase 2（1〜2週間）: スコア40未満は「統合」または「抜本リライト」
40点未満は **救えるかどうかを記事単位で判断**する。

- **救えるケース**: トピックの検索意図はあるが、文字数・出典・監修が不足 → **抜本リライト**（既存URLを保ち、本文の80%以上を書き換える。URL変更しないので評価リセットを避けられる）
- **救えないケース**: 似たトピックの記事が複数ある／検索意図がそもそも薄い → **記事を統合**（弱い側を301リダイレクトで強い側に寄せる。これによりリンク評価とインデックス資産を統合先に集約できる）
- 統合先がない、かつ救えない、という稀なケースのみ Phase 5 で noindex を検討

### Phase 3（2〜4週間）: スコア40-59は「本文拡充＋内部リンク強化」
40-59点は **削除や統合の対象ではなく、磨けば光るゾーン**。

1. 不足している項目をスコアレポートの `suggestions` 列から特定（例: 内部リンク0本、外部出典なし）
2. 公的出典（厚労省・総務省・国民生活センター・自治体）を最低1本追加
3. 関連記事3本への内部リンクを追加（ピラー記事・チェックリスト・はじめかた）
4. オリジナル画像 or 図解を1点追加（HCS対策の「First-hand experience」シグナル）
5. 監修者付与（生前整理系=村上充恵様 / 専門家領域は該当士業）

### Phase 4（4〜8週間）: GSCで実パフォーマンスを観察
リライト完了後、**最低4週間はGSCで以下を観察**：
- impressions の立ち上がり（0 → 1桁 → 2桁 への遷移）
- 平均掲載順位（初期は40-80位、徐々に20位圏内へ）
- ターゲットKWでのインデックス確認（`site:fureaino-oka.com/articles/{contentId}`）

`npm run gsc:report` を週次で回し、`logs/gsc/` に蓄積していく。**この期間に「リライト前後の impressions 推移」を contentId 単位で記録するのが M&A 説明資料の価値ある一次データになる。**

### Phase 5（8週間以降）: それでも0impのページのみnoindex検討
Phase 1〜4 を経ても、**8週間連続で impressions=0** のページがあれば、その時点でようやくnoindex候補となる。判断順序:

1. まず `<meta name="robots" content="noindex">` を入れる（サイトマップからは外す）
2. 1ヶ月後、GSCで「インデックス削除済み」を確認
3. 完全に評価を切り離すなら、その後で削除＋410 Goneを返す
4. **同時に「なぜそのKWが流入を生まなかったか」を学習として `docs/seo-keywords.csv` の負の知見欄に記録**

ここまで来た記事のみが、本当の意味で「失敗作」である。実数は100記事中おそらく10〜20本程度に収まるはずだ。

## 3. HCS（Helpful Content System）への正しい対処

「即noindex」を考える本当の動機は、HCSペナルティへの恐怖だろう。だがHCSが嫌うのは**「数の多さ」ではなく「人間に役立たない感」**である。対処は以下に集約される：

### 3-1. AI生成感を消す（First-hand experience シグナル）
- **実体験**: 「私の祖母が亡くなった時、実家の片付けに3週間かかった」のような具体エピソード
- **独自データ**: ふれあいの丘が持つ`municipalities.json`の自治体データを根拠に「全国1,741自治体のうちXX%が補助金制度を持つ」など、他サイトに無い数字を提示
- **専門家コメント**: 村上充恵様（総合監修）のコメントを記事内に直接引用形式で挿入

### 3-2. 監修者明示（既に対応中の領域を全記事に拡げる）
- 生前整理／介護離職予防／親子コミュニケーション系 → 村上充恵様
- 相続税／不動産売却／解体／登記 → 該当士業（**村上様は付けない**。`docs/SUPERVISOR_PLACEMENT_AND_PROFILE.md` の領域分離ルール厳守）
- 大久保（運営者）は**生前整理アドバイザー2級**の表記に統一。**税理士・弁護士の領域には踏み込ませない**

### 3-3. 似たトピックの統合（Phase 2 と連動）
HCSは「同じテーマで薄い記事が10本」を最も嫌う。例えば「実家じまい 業者」「実家じまい 費用」「実家じまい 流れ」が別記事になっているなら、**1本のピラー記事に統合し、それぞれを目次（H2/H3）として包含する**のが正解。これでリンクジュースが集中し、HCSの「網羅性」シグナルにも応える。

### 3-4. CTA配置と本文の分離（既存ルールの再確認）
- LINE誘導CTAは**記事本文(body)に含めない** — テンプレ側 `ArticleLineCTATop` / `ArticleFooterCTA` で固定表示
- これはHCS対策（広告/CTAばかりの記事は低評価）と、運用効率（CTA文言変更を一括できる）の両方の理由

## 4. ふれあいの丘 固有の状況分析

### 4-1. GSC（過去90日）から読み取れる事実
- サイト全体 CTR=0.39% は確かに低い（業界目安は2-3%）
- ただし `articles/*` 配下は **impressions=11, clicks=0** — そもそも検索結果に表示されていない（CTR以前の問題）
- 一方 `tools/tax-simulator` は **CTR=3.32%** — 一定の流入と意図合致を獲得済み
- これらは「100記事が悪い」のではなく「100記事がまだGoogleに認識されていない」ことの証左

### 4-2. なぜ即判断が禁物か
週末に100記事を投入したのが直近の出来事だとすれば、**そもそもクローラが100記事を全て発見し終わっていない可能性が高い**。`/sitemaps/articles/sitemap.xml` での通知 → Googlebot のクロール → インデックス登録には数週間かかる。

以下を**まず確認**してから判断すべき:
1. GSC「ページ」レポートで `articles/*` のインデックス済み数を確認（投入100に対していくつ？）
2. `npm run index:run` で Indexing API 経由のクロール促進を実施したか
3. `/sitemaps/articles/sitemap.xml` がGoogleに送信され、Last read date が直近か

これらが未実施なら、**noindex の前にやることがある**。

### 4-3. 「100記事のうち実流入0」の正しい解釈
- **誤った解釈**: 「100記事すべてが失敗。即削除すべき」
- **正しい解釈**: 「100記事はまだGoogleの評価サイクルに入っていない。最低90日待ち、それまでに品質スコアリングで内部改善を進める」

GA4と GSC の両方で**コンテンツ単位の `revisedAt → impressions` 立ち上がりカーブ**を観察するのが、HCSへの最も誠実な対応である。

## 6. やってはいけないこと（NG リスト）

- 100記事を一括 `noindex` ／一括削除
- 一括で robots.txt から articles を Disallow（クローラが二度と来なくなる）
- URL構造の大幅変更（評価リセットを招く）
- 「とりあえず公開した」記事の放置（HCSの最大の敵は「メンテされていない感」）
- 大久保名義での税務・法務記事の量産（YMYL×無資格 = HCS的に最悪の組み合わせ）
- 監修なし記事の追加公開（既に100本ある状況でさらに監修なしを足すと比率が悪化）

## 7. 想定 Q&A

**Q. でも他のSEO業者は「悪い記事はすぐ消せ」と言うのでは？**
A. それは「サイト全体の数%」を消す話。今回は「100/総記事数」が大半を占める状況で、ドメイン全体への影響が桁違いになる。同じアドバイスを適用してはいけない。

**Q. 8週間も待つ余裕がない（M&Aを急いでいる）**
A. 急ぐほど慎重になるべき。買い手のDDは過去6ヶ月のGSC/GA4データを見る。**「リライトして改善した記録」は資産になるが、「大量noindex」は負債になる**。8週間で評価額数百万円の差が出る判断と心得る。

**Q. AI生成記事だとバレるのが怖い
A. バレるかどうかではなく**「読者の役に立つか」がHCSの判定軸**。AI生成でも実体験・独自データ・専門家監修が乗っていれば評価される。逆に人間が書いても薄ければ落とされる。怖がるべきは生成方法ではなく、品質の薄さである。
