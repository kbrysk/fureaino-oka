## 1. 現状の title・meta description テンプレート（手順1の結果）

### 生成箇所
- **ファイル**: `app/area/[prefecture]/[city]/subsidy/page.tsx`
- **関数**: `generateMetadata`（69行目〜）

### 変更前の title テンプレート
- 基本形: `【${currentYear}年度版】${cityName}の空き家・解体補助金｜${maxLabel}・申請条件・手順を図解`
- 40文字超: `【${currentYear}年度版】${cityName}の空き家・解体補助金｜${maxLabel}`
- 50文字超: 上記を先頭49文字で切って「…」付与
- `maxLabel` = `extractMaxAmount(data.subsidy?.maxAmount)` の結果、または **「最大補助金あり」**（金額なし時）

### 変更前の meta description テンプレート
- `${cityName}で使える空き家・解体補助金を${currentYear}年度最新情報でまとめています。補助金額・申請条件・必要書類・申請期間を図解で解説。無料で解体費用の見積もりも取得できます。`
- 120文字超時は119文字で切って「…」付与

### 補助金データの取得元
- **メイン**: `getMunicipalityDataOrDefault(prefecture, city, fallback)` → **app/lib/data/municipalities.json**  
  - `subsidy.hasSubsidy`, `subsidy.name`, `subsidy.maxAmount`, `subsidy.conditions` 等
- **上書き用**: `getAreaContent(prefecture, city)` → **data/area-contents/[pref]/[city].json**（AreaContentData.subsidyInfo）
- メタ用の補助金情報は `areaContent?.subsidyInfo ?? (data.subsidy から構築)` で決定し、`extractMaxAmount(subsidyInfo?.maxAmount ?? data.subsidy?.maxAmount)` で「最大○万円」を抽出

## 3. 広島市の対応方針（手順5の判断）

**採用方針: B）現状のまま title 最適化のみ行う**

### 理由
- 当該URLは **解体補助金** 専用ページであり、検索意図の中心は「補助金・申請条件」である。
- 「空き家処分 広島市」「空き家整理 広島市」で表示されるのは、Google が 空き家＋広島市 の関連性で補助金ページを出している可能性が高く、**処分・整理クエリに最適なのは遺品整理・粗大ゴミ等の別ページ**である。
- title/description に「処分・整理」を入れると、補助金を探しているユーザーへのメッセージが弱くなり、CTR が伸びにくい。
- 今回の **「【年最新】市区町村 空き家解体補助金 最大○万円」形式** に統一することで、補助金検索意図との一致が明確になり、広島市ページも含めて CTR 改善が期待できる。

### A を取らない理由
- 処分・整理のニーズを title/description に含めると、ページの主題が「補助金」からブレる。
- 処分・整理で流入したユーザーは、補助金より「業者・手順」を求めており、別コンテンツ（cleanup/garbage 等）で対応する方が適切。
