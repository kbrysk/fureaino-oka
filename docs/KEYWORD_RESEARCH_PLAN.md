# メインコンテンツ KW設計図 ＆ ラッコキーワード抽出指示書

> 目的：4本のヒーロー・ピラー（網羅型3,000〜5,000字）を書く前に、「どのKWで・どんな内容を狙うか」を固める。
> 役割分担：本書のシード語を大久保さんがラッコキーワードにかけ、関連語＋月間ボリュームを書き出す → 私が台帳 `content/pipeline/keywords.csv` に統合し優先度確定 → ブリーフ → 執筆。
> 関連：`docs/MAIN_CONTENT_STRATEGY.md`（戦略）、`SEO_CONTENT_AUTOMATION_ARCHITECTURE.md`（量産）、競合・SERP調査（みんなの遺品整理／クラッソーネ／相続会議）。

---

## 0. 設計方針（ピラー＆クラスタ）

- **ピラー（ハブ）＝網羅型 3,000〜5,000字**。そのテーマの「決定版」を目指し、内部リンクでクラスタ記事を束ねる。
- **クラスタ（個別）＝2,000〜2,500字**。1記事1検索意図。ピラーへ内部リンクで権威を集約。
- **監修**：生前整理・家族・進め方は大久保（2級）→村上様に切替。相続税・不動産・解体・登記は専門家確保まで「一般情報＋専門家相談」（総合監修クレジットは付けない）。
- **CV先**：F1共感＝診断ツール/無料PDF（メール獲得）、F2処分＝買取/不用品見積もり、F3不動産＝査定アフィ。
- **競合回避**：法務・税務の正面戦（相続会議）と不動産査定の広告戦（SUUMO等）は避け、「自治体別データ×家族・感情論点×総額シミュレーター」の隙間を取る。

## 1. 4ピラーのKWマップ

各ピラーの「シード語」をラッコにかける。`[要ラッコ]`＝ボリューム取得対象。サブKW候補は私の想定（ラッコ結果で差し替え・追加）。

### P1：実家じまい完全ロードマップ（中央ハブ・最重要）
- **役割**：親の施設入居→片付け→補助金→売却→相続を時系列で1本に統合。全クラスタ・全CVの分岐点。
- **監修**：大久保（進め方）。相続・不動産部分は一般情報＋専門家相談。
- **シード語（ラッコ投入）**：`実家じまい` / `実家じまい とは` / `実家じまい 流れ` / `実家じまい 進め方` / `実家じまい 費用` / `実家じまい いつから` / `実家じまい 後悔` / `実家 片付け どこから`
- **想定サブKW/論点**：実家じまい チェックリスト、実家じまい 期間、実家じまい 親が生きてる、実家じまい 遠方、実家じまい やることリスト
- **CV**：診断ツール（実家じまい力診断）＋無料PDF。各工程からP2/P3へ送客。

### P2：遺品整理・実家片付け 費用ガイド ＋ 総額シミュレーター
- **役割**：費用相場の決定版＋シミュレーターで送客の心臓。自治体データ（`municipalities.json`）で独自性。
- **監修**：大久保（片付けの進め方）。費用は一般相場情報。
- **シード語**：`遺品整理 費用` / `遺品整理 費用 相場` / `実家 片付け 費用` / `生前整理 費用` / `遺品整理 費用 一軒家` / `不用品回収 費用 相場` / `ゴミ屋敷 片付け 費用`
- **想定サブKW/論点**：間取り別（1R/1LDK/3LDK/一軒家）、遺品整理 安く する方法、遺品整理 自分で、遺品整理 業者 選び方、追加料金 トラブル
- **CV**：不用品/遺品整理 見積もり送客。シミュレーター→査定誘導。

### P3：空き家・実家の売却と解体補助金（自治体pSEOハブ）
- **役割**：既存の `/area/.../subsidy` を束ねるハブ。高単価の不動産査定送客。
- **監修**：不動産＝宅建士／税務＝税理士（確保まで一般情報＋相談）。**総合監修クレジットは付けない領域**。
- **シード語**：`実家 売却 流れ` / `空き家 解体 費用` / `空き家 解体 補助金` / `相続 空き家 売却` / `実家 売る タイミング` / `空き家 固定資産税` / `相続空き家 3000万円控除`
- **想定サブKW/論点**：自治体名×解体補助金（pSEO）、空き家 維持費、更地 デメリット、相続登記 義務化、譲渡所得税（制度概要のみ）
- **CV**：不動産一括査定アフィ（高単価）＋解体見積もり。

### P4：親に生前整理を切り出すには／家族会議の進め方 ★第1弾 執筆済み
- **役割**：感情・意思決定の上流集客。競合が手薄で村上様/大久保の専門ど真ん中＝E-E-A-T差別化。
- **監修**：大久保（2級）→村上様。
- **シード語**：`実家 片付け 親 説得` / `生前整理 親 嫌がる` / `親 物を捨てない 心理` / `実家じまい 親 反対` / `生前整理 切り出し方` / `きょうだい 遺品 もめる` / `親 終活 話し方`
- **想定サブKW/論点**：親 片付けない ストレス、実家 片付け 親 喧嘩、家族会議 進め方、遠距離 親 介護 片付け
- **CV**：診断ツール→リスト化→無料PDF。
- **状況**：ハブ記事 `content/pipeline/oya-settoku/` 執筆済み（QA満点）。このシードのクラスタを今後拡張。

## 2. ラッコキーワード 抽出指示書（大久保さん作業）

### やること
1. [ラッコキーワード](https://related-keywords.com/) を開く。
2. 上記**シード語（P1〜P4の太字シード、計約30語）**を1つずつ入力し「関連キーワード取得」。
3. **月間検索ボリューム**を表示（ラッコの「検索Vol.チェッカー」/Googleキーワードプランナー連携。有料機能の場合あり）。
4. 各シードの結果を**CSVでダウンロード**（ラッコは候補語一覧のCSV出力に対応）。

### 書き出してほしい項目（最低限）
| 列 | 内容 |
|---|---|
| keyword | 関連キーワード |
| volume | 月間検索ボリューム（取れれば） |
| seed | どのシードから出たか（P1/P2/P3/P4のどれか分かるとベター） |

CPC・競合性も取れれば歓迎（収益性・難易度の判断に使う）。

### 戻し方
- CSVを `content/pipeline/rakko/` に置く（ファイル名は何でもOK。例 `p1-jikkajimai.csv`）か、内容を貼り付けてください。
- 私が `content/pipeline/keywords.csv`（台帳）に統合し、検索意図・CV先・クラスタ・監修区分・優先度を付与します。

> ボリュームが取れない（無料枠の制限等）場合でも、**関連語の一覧だけ**でも大きな価値があります。その場合は私がWebSearchで補完しつつ、競合SERPの強さで優先度を推定します。

## 3. 優先度の付け方（ボリューム取得後に確定）

次の式でスコア化し、ピラー→クラスタの執筆順を決める：
- **優先度 = 検索ボリューム × CV近接度（F3>F2>F1の収益性も加味） × 勝てる見込み（競合の弱さ・自社独自データの有無） ÷ YMYL難易度（監修要否）**
- 立ち上げ期は「**大久保監修で書けて（生前整理・家族・進め方）× 競合が弱く × CVに繋がる**」を上位に。P4クラスタとP1から。

## 4. 次工程（このあとの流れ）

1. （大久保さん）ラッコでシード語のボリューム取得 → CSV共有。
2. （私）台帳統合・優先度確定 → 各ピラーのブリーフ作成（`serp-outliner`相当：上位SERPの必須見出し＋独自切り口）。
3. （私）3,000〜5,000字でピラー執筆 → `qa-lint` 合格 → 大久保監修で公開準備。
4. クラスタ記事を半自動パイプラインで監修付き量産。
