## 0. 背景と問題意識

### 現状（2026-06-01 時点）

- **実装済アフィ**:
  - ノムコム（野村不動産ソリューションズ） — `a8mat=4AXE4D+D2CGOI+5M76+BX3J6` / 不動産一般エリア
  - ワケガイ — `a8mat=4AXDCK+E6TXTE+5J56+5YRHE` / 訳あり物件・古家・再建築不可
  - アールクリーニング — `a8mat=4AXE4D+BUADWY+4X26+NTRMQ` / 首都圏（東京・神奈川・埼玉・千葉の一部）
  - ECOクリーン — `a8mat=4AXDCK+E68I7M+36X8+15OK2A` / 全国（首都圏外フォールバック）
- **直近成果**: 2026/04 にノムコム経由で **5,000円成果1件**確定。

### 課題

1. **ノムコムは「営業エリア外」否認リスク**: 野村不動産ソリューションズは大手だが営業エリアが限られ、地方の市区町村ページから流入したリードが「対応エリア外」で否認される懸念。
2. **全国 pSEO との不整合**: ふれあいの丘は `app/area/[prefecture]/[city]/(subsidy|cleanup)` で全国の市区町村ページを `generateStaticParams` で量産している。**全国対応**の査定アフィを並列で持ち、エリア別に切り替える必要がある。
3. **解体補助金ページの収益化が弱い**: `/area/[prefecture]/[city]/subsidy` は解体補助金情報の精度で集客しているが、現状は「ノムコム」（売却査定）への導線しかなく、**解体一括見積もり**アフィを置けば CVR が上がる読者層が抜けている。
4. **遺品整理アフィの未開拓**: ECO/アールクリーニングは「片付け業者直販」だが、「遺品整理110番」「みんなの遺品整理」等の**業者紹介ポータル**は別カテゴリで成果が立てやすい。

### 本ドキュメントのゴール

- A8.net 内で**ふれあいの丘と相性が良いプログラム候補を網羅的にリストアップ**。
- 各プログラムの**成果報酬目安・否認条件・全国対応可否**を整理し、即追加すべき2社を特定。
- 既存 `RealEstateAppraisalCard.tsx` に**最小改修**で組み込む A/B テスト実装案を提示。

> **重要な但し書き**: A8.net のプログラム名・成果報酬・確定率は変動するため、本書の数値は **2026年5月時点の公開情報・業界平均からの推定**です。実際の提携前に **A8 管理画面で「プログラム検索」→ 詳細条件を必ず再確認**してください。本書では確認できない箇所に「※2026年5月時点情報・要再確認」を付記。

## 6. A/Bテスト設計

### 6-1. 設計の前提

- **既存ノムコム成果を毀損しない**。ノムコムは確定実績がある。
- エリア別で**ノムコム or イエウール（または他全国対応）を切り替え**、ノムコム否認リスク領域だけフォールバックさせる。
- 計測は既存 `app/api/affiliate/appraisal/route.ts` の `type` パラメータを拡張して経路ログを残す。

### 6-2. 切り替えロジック案（最小実装）

```ts
// app/components/RealEstateAppraisalCard.tsx 改修案
const NOMU_AREA = new Set<string>([
  // ノムコム営業実績の濃いエリア（首都圏・関西・名古屋・福岡等）
  // 都道府県コード or cityId のプレフィックス（"13", "14", "27", "28", "23", "40" 等）で判定
]);

type AppraisalVariant = "nomu" | "ieuru" | "wakegai";

function useVariant(cityId: string, localRiskText: string | null | undefined): AppraisalVariant {
  if (typeof localRiskText === "string" && WAKEGAI_KEYWORDS.test(localRiskText)) return "wakegai";
  const prefCode = cityId.slice(0, 2);
  return NOMU_AREA.has(prefCode) ? "nomu" : "ieuru";
}
```

- リダイレクト先は `/api/affiliate/appraisal?area=<cityId>&type=<nomu|ieuru|wakegai>` で集約。
- `route.ts` に `IEURU_URL` フォールバックを追加（パターンは既存と同じ）。

### 6-3. 想定エリア分割（初期）

| カテゴリ | 対象都道府県 | 推奨表示 |
| --- | --- | --- |
| 大都市圏（ノムコム強い） | 東京・神奈川・千葉・埼玉・大阪・京都・兵庫・愛知・福岡 | **ノムコム** |
| 地方主要都市 | 北海道（札幌）・宮城・広島・新潟等 | **ノムコム or イエウール A/B**（2週間で勝者決定） |
| 地方・郡部 | 上記以外 | **イエウール**（全国対応・否認リスク低） |
| 訳あり物件キーワード検出時 | 全国共通 | **ワケガイ**（既存ロジック維持） |

### 6-4. 期待効果（仮説）

- **CVR**: ノムコム否認エリアでイエウールに切替 → 否認率 -30%、確定報酬 +15〜25%。
- **CTR**: ボタン文言を「全国対応」「最大6社比較」に変更 → 地方ユーザーの安心感で +5〜10%。
- **収益期待**: 月間PV 100,000、CTR 1.5%、CVR 3%、平均報酬 2,000円、確定率 65% と仮置きすると、追加 **約 60,000円/月** の上振れ期待（保守的見積もり）。

### 6-5. 計測

- `app/api/affiliate/appraisal/route.ts` の `console.info` ログを **Vercel Logs → BigQuery/Logflare** へ送り、エリア別・variant別のクリック数を毎週集計。
- A8 管理画面の確定率と突き合わせて2週間ごとに variant の重みを調整。

## 9. リスクと注意点

- **法務（CLAUDE.md 遵守）**:
  - 押し買い・不当表示系（「絶対高額査定」「最高額保証」等）プログラムは**除外**。本書の候補はすべて適法な一括査定/見積もりサービスのみ。
  - 「査定価格は適正評価」「保証は業者次第」の文言を維持し、断定的表現を避ける。
- **YMYL観点**:
  - 査定・解体・遺品整理は金銭判断を伴うため、CTA は「無料相談・見積もり比較」までに留め、「絶対売却すべき」等の煽りは禁止。
- **重複申込否認**:
  - 同一ページに複数の一括査定を並べると、ユーザーが両方申し込んで**両方否認**されることがある。**1ページ1プログラム**を厳守し、A/Bは表示時に二者択一とする。
- **インプレッション計測**:
  - A8 のインプレッション画像URLは `www<N>.a8.net` の `N` がプログラムごとに異なる。コピペミスを避けるため定数化＋環境変数化を徹底。
