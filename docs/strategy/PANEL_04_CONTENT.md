## 0. 結論（先に断言）

**「ふれあいの丘」がコンテンツで勝てるのは、"取引KW（業者・買取・費用比較）"ではなく、"意思決定KW（実家じまいロードマップ・親への切り出し・空き家×補助金×固定資産税の判断）"だ。** 前者は東証上場の買取大手（バイセル）・比較ポータル（みんなの遺品整理＝LIFULL／くらしのマーケット）・士業監修メディア（相続会議）がCPC¥1,000超の物量で支配する消耗戦で、生前整理2級の個人監修では構造的に勝てない。後者は「親の施設入居→片付け→補助金→売却→相続」を**時系列で1本に束ねた当事者目線のロードマップ**と、**全国自治体データ（`municipalities.json`）×独自シミュレーター×大久保の実体験**という、ポータルにも士業にも作れない情報利得（information gain）で取りに行く。**収益はこのロードマップの"出口"＝空き家・実家売却の不動産査定アフィ（¥5,000実証ルート）に一点集中させ、取引KWの記事は買取/不用品アフィの送客台として最小限に保守する。** これが唯一の勝ち筋である。

CV先（`cv_destination`）の現状分布：lead_pdf 28本 / fudosan_satei 7本 / 買取系（kaitori/ihin/fuyohin）18本 / souzoku 6本。**問題は、収益実証済みの唯一ルートが「不動産査定アフィ¥5,000」なのに、それに紐づく記事が7本しかないこと。** 配線を意図とCVで整合させる。

| 検索意図タイプ | 代表KW | 整合するCV | 設計方針 |
|---|---|---|---|
| **資産処分・売却の意思決定** | 空き家処分／実家 売却／相続不動産／固定資産税6倍 | **fudosan_satei（¥5,000実証）** | 最優先で増床。記事末＋本文中盤に査定CTA。「持ち続ける固定資産税 vs 売却」の比較で査定へ橋渡し |
| **「最初の一歩」が踏み出せない** | 生前整理 何から／終活 いつから／親への切り出し | **lead_pdf（メール獲得）** | 調査で最大の壁が「最初の一歩」。**チェックリストPDF**で着火。Zero-Party Data蓄積 |
| **費用が知りたい（行動前）** | 実家じまい費用／遺品整理 費用／解体費用 | **シミュレーター→査定 or PDF** | tax-simulatorの成功を横展開。費用算出後に査定/見積もりへ |
| **不用品を換金したい** | 着物買取／骨董／遺品 買取 | 買取アフィ（kaitori/ihin） | 消耗戦領域。送客台に徹し**深追いしない** |
| **手続き・段取り** | 親が亡くなったら／口座凍結 | lead_pdf（チェックリスト） | リード獲得。査定には繋げにくいのでPDF一択 |

**整合の要点：** ①査定CV記事（P3空き家・実家売却）を7→15本に倍増。②全lead_pdf記事のPDFを「クラスター別チェックリスト」に差し替え（汎用PDFは離脱）。③買取系は新規を作らず既存維持。

## 3. CV直結トピッククラスター設計図

勝てる領域に絞り、**ピラー（網羅・権威集約）＋クラスター（検索意図の枝）＋CV出口**の三層で設計。

### クラスターA：実家じまい完全ロードマップ（ピラー＝最重要ハブ）

  ├─ 親の施設入居/介護の判断 ─→ P4へ
  ├─ 片付け・遺品整理の段取り ─→ C6ピラー（保守）
  ├─ 費用の全体像 ─→【総額シミュレーター】──CV: 査定/PDF
  ├─ 空き家になった実家の選択肢 ─→ クラスターB（売却×補助金）★CV最強
  └─ 相続・名義 ─→ C11チェックリスト（深入りしない）
  CV: 中盤=費用シミュレーター / 末尾=無料ロードマップPDF（メール）
```

### クラスターB：空き家×実家売却×補助金（CV最強・収益エンジン）
```
[PILLAR] 空き家になった実家、どうする？4つの選択肢（売る/貸す/解体/管理）
  ├─ 空き家解体費用＋【解体費シミュレーター】
  ├─ 空き家の解体補助金（自治体別＝municipalities.json）★情報利得
  ├─ 特定空き家・固定資産税6倍／京都市空き家税2026 ★鮮度
  ├─ [pSEO] 〇〇市 空き家 補助金 ×全国 ──唯一の実クリック源
  └─ 実家を売却する流れ・不動産査定 ─→★fudosan_satei（¥5,000）
  CV: 全枝から「固定資産税を払い続ける vs 売る」比較→査定CTA一点集中
```

### クラスターC：親への切り出し・家族の感情（差別化最大／リード源）
```
[PILLAR] 親に終活・片付けをどう切り出す？（大久保の現場知見）
  ├─ 物を捨てられない高齢の親への接し方
  ├─ きょうだい間の不公平・トラブル回避（家族会議の進め方）
  ├─ 介護離職を防ぐ段取り
  └─ 親の財産・通帳の把握（PII非保存・一般論）

## 4. 独自性（information gain）の作り方 — AI Overview時代に引用される要素

2026年、AI Overviewは検索の6割超で表示され従来1位の上に出る。引用されるのは**一次情報・独自データ・構造化（表/箇条書き）・E-E-A-T**。ポータルや士業の"一般論の焼き直し"では引用されない。本サイトの武器は4つ：

1. **全国自治体データの分析（`municipalities.json`）＝唯一無二の一次データ。**
   「解体補助金の上限額 全国TOP20」「粗大ごみ料金が高い/安い自治体ランキング」「補助金がある自治体の割合」など、**JSONを集計した独自統計表**を記事に埋め込む。ポータルは"リンク集"止まりで作れない。AI Overviewが好む「表＋出典付き一次データ」そのもの。
2. **大久保亮佑（生前整理2級）の現場コメント＝Experience（経験）の実装。**
   各ピラーに「現場で実際にあった事例」ボックスを1つ以上。例：「捨てられない親には"全部捨てる"でなく"1箱だけ残す"提案が通った」。AI Overviewと品質評価が重視する一次経験。**ただし税務・法務には踏み込まない（2級の専門範囲厳守）。**
3. **独自シミュレーション結果の公開。** tax-simulatorが唯一CTR3.32%で機能している事実＝「ツール由来の独自数値」が刺さる証拠。「築40年・〇〇市・2階建てを更地化した場合の固定資産税の変化」などの**算出結果サンプルを記事内に表で提示**し、ツールへ送客。
4. **当事者の時系列ナラティブ（統合ロードマップ）。** 「施設入居→片付け→補助金→売却→相続」を1本で扱う構造自体が情報利得。断片化した競合に対する網羅性の優位。

> 各記事チェック：「この記事にしか無い数字・経験・図表が**最低1つ**あるか？」無ければAI Overview時代は無価値。リンター（現平均84/100）に**information_gain項目を追加**することを推奨。

## 7. コンテンツ × ツールの融合（最大の武器化）

tax-simulatorが**唯一CTR3.32%で機能している事実は最重要シグナル**。「読む」より「自分の数字を出す」体験がCVを生む。ツールをコンテンツの中核に据える。

1. **記事内インライン埋め込み：** 関連ピラー（空き家・実家じまい費用）に、記事を読み終える前の中盤位置でシミュレーターを埋め込む。離脱前に"自分ごと化"。
2. **算出結果→査定/PDFのブリッジ：** シミュレーター結果画面に「この固定資産税を払い続けますか？売却査定を見る」（査定CV）／「結果をPDFで保存（メール獲得）」を必ず併置。**ツールをCVファネルの入口に**。
3. **ツール由来の独自データを記事化：** 「シミュレーター利用者の入力傾向から見た、〇〇な実家の固定資産税中央値」など、ツールが生む匿名集計を記事の information gain に還流（PII非保存厳守）。
4. **診断ツールの拡張：** 空き家リスク診断・相続シミュレーターを、クラスターB/Aの各ピラー末尾に「次の一歩」として配置。**全ピラーに最低1ツール**を原則化。

## まとめ：コンテンツ戦略の勝ち筋（1段落・断言）

**勝ち筋は「取引KWを捨て、意思決定KWに資源を一点集中する」こと。** 遺品整理・買取・業者比較は上場企業とCPC¥1,000超のポータルが支配する消耗戦であり、生前整理2級の個人監修では1円も生まない領域だ。代わりに、`municipalities.json`の全国自治体データ・大久保の現場経験・独自シミュレーターという**ポータルにも士業にも複製不可能な情報利得**を束ね、「実家じまいロードマップ（クラスターA）」「空き家×売却×補助金（クラスターB＝収益心臓部）」「親への切り出し（クラスターC＝差別化）」の3クラスターに集中する。収益は唯一実証済みの**不動産査定アフィ（¥5,000）**に全内部リンクの重心を寄せ、リードはクラスター別チェックリストPDFでZero-Partyを蓄積する。AI Overview時代に引用されるのは"自社データ・一次経験・構造化表"であり、それを持つのは大手ではなく**当事者目線でデータを束ねる本サイトだけ**。撤退の勇気と一点集中が、エグジット価値を作る。
