# 費用ゼロ・自動化前提の被リンク獲得戦略 ― 3視点 独立分析 → 止揚（2026-06）

> 課題: DR 21。被リンクがサイト最大のボトルネック。
> 制約: **費用ゼロ**（PR TIMES等の有料配信3万円/月は使わない）。**人手は最小、自動工数（Claude Code）は無制限**。
> 目的: 関連性の高いサイトからの**編集的に獲得した（earned）**被リンクを、再現可能な"仕組み"として構築し、年内3,000万円エグジットの資産価値に変える。
> 凡例: 自動化度 ◎=ほぼ全自動 / ○=発見・下書き自動＋人間は送信のみ / △=人間の関係構築が主

---

## 0. 結論先出し（TL;DR）

**勝ち筋は1つ**――「**Citable Asset Engine（引用される資産エンジン）**」。
1,726自治体の独自DBと無料ツールを"**引用したくなる一次資産**"に変え、**Claude Code を「発見→評価→下書き」の無料エンジン**として回す。人間は「**送信ボタンと関係構築の最後の1マイル**」だけ。

費用ゼロで最も効く順に、今日から着手できる3手：
1. **リンクなし言及の回収（Unlinked Mention Reclamation）** ― すでに自分の名前が出ているのにリンクが無いページを見つけ、リンク化を依頼する。最も成功率が高い"刈り取り"。（既存 `mention-watch.mjs` をClaude Code版で無料運用）
2. **データ資産の引用元化** ― 補助金DBを「全国調査レポート」ページにし、`Dataset`構造化＋"引用可能な統計"として出す。記者・ブロガーが引用＝自然リンク。（既存 `/data` `data-reports/` を拡張）
3. **.lg.jp/.go.jp への"内容確認"先行アプローチ** ― 自治体に「リンクください」ではなく「住民向け資料の内容確認」を依頼。公的ドメインからの最高品質リンクを誠実に狙う。（既存 `municipality-outreach.mjs` + `/public-sector`）

## 1. 三者 独立分析（それぞれ冷徹に）

### 視点① SEO／被リンク獲得スペシャリスト
- **現状認識**: DR 21・参照ドメインが極小。コンテンツ量に対してリンクが伴わず、これが順位の天井。
- **質 > 量の原則**: 関連性の高い**dofollowリンク10本** ＞ ディレクトリ/無料PRの**nofollow 1,000本**。後者はほぼ価値が無く、低品質ディレクトリへの大量登録は**スパムfootprint**を残す。
- **無料PR配信サイトの冷徹な評価**: ValuePress無料プラン（到達10媒体程度）・DreamNews等は**リンクが基本nofollow**で被リンク価値はほぼゼロ。使うなら「**指名検索の種まき・サイテーション・referral**」目的に限定し、被リンク施策の本線にはしない。
- **本命チャネル（費用ゼロで関連性高）**:
  1. **Unlinked Mention Reclamation**（言及済・リンク無しの回収）＝最高ROI
  2. **Linkable Assets**（独自データ・無料ツール・図表）への自然リンク
  3. **リソースページ／リンク集ターゲティング**（"役立つサイト""関連リンク"ページへの掲載）
  4. **壊れリンク回収（Broken Link Building）**＝廃止された他サイトの参照先を自分の該当ページに置換提案
- **アンカー戦略**: 完全一致の商用KW（例「不動産査定」）を狙わない。**ブランド名・URL・自然文**アンカーで多様性を保つ（過剰最適化はSpamBrainの標的）。

### 視点② Googleアルゴリズム最終責任者
- **大前提**: リンクは"作る"ものではなく**編集者が自発的に張る（earned）**もの。リンクスパムポリシー＋SpamBrainは、**自動化された大量アウトリーチ・相互リンクの組織化・完全一致アンカーの操作**を検出し**無効化**する（ペナルティ以前に"効かなくする"）。
- **Googleが報いたいもの = Information Gain（情報利得）**: 既存のどこにも無い**一次データ**。本サイトの**1,726自治体の横断集計**は、まさにGoogleが引用・評価したい類の資産。これを構造化（`Dataset`/`ItemList`/統計の断定）して"機械可読な引用元"にすれば、AI Overview/AI検索でも引用されやすい（GEO効果）。
- **nofollow至上主義を捨てよ**: nofollow/UGCリンクや**リンク無しの言及**も、Googleはエンティティの信頼・実在性（E-E-A-T）の文脈シグナルとして扱う。dofollowだけを追うと機会を逃す。
- **最強リンク**: `.go.jp` `.lg.jp` `.ac.jp` `.or.jp` からの**自然な参照**。ただし**依頼の仕方を誤ると逆効果**。自治体への初回接触は「掲載依頼」ではなく「公共資料としての内容確認」に徹するべき（既存設計は正しい）。
- **レッドライン（無効化/手動対策リスク）**: 大規模リンク交換、有料リンク、PBN、コメント/フォーラムスパム、同一文面の大量送信、サテライト乱立、自作自演プロフィールリンクの量産。

### 視点③ シリアルアントレプレナー（ノンペイド・グロースハック × 複数Exit）
- **レバレッジ＝人手を使わない**: 1通ずつ手で送るのは負け。**「発見・評価・下書き」を自動化**し、人間は"最後の承認と関係づくり"だけに資源を集中。
- **資産として productize**: 補助金DBを**四半期ごとの「全国調査データドロップ」**に定例化（季節性: 年末の片付け需要、確定申告期、相続シーズン）。1度作れば毎期"ニュースになる素材"を再生産でき、**リンクが複利で積み上がる**。
- **無料ツール＝複利エンジン**: 「あなたの市の空き家解体補助金が分かる」診断・検索ツールは**本質的にリンクされやすい**（人は便利な無料ツールに張る）。**埋め込み（embed）＋出典リンク**を付ければ、他サイトが勝手に設置してリンクが増える（engineering as marketing）。
- **分配チャネルはタダで存在する**: 記者・編集者は**X / note / 業界ニュースレター**にいる。日本の**無償の記者マッチング（P-NEWS無償プラン／PRONEのネタ募集・記者クラブ情報）**で取材ネタとして"一次データ"を出せば、編集記事＝高品質リンクになり得る。
- **Exit観点の核心**: 買い手は「**今あるリンク数**」より「**毎月リンクが増える再現可能なエンジンがあるか**」を評価する。**月次の参照ドメイン増加グラフ**こそが評価額を押し上げる。だから"単発の獲得"でなく"仕組み"を作る。
- **過剰設計を戒める**: 最初から全部作らない。**1資産（データレポート）×1チャネル（言及回収）**で回し、数字が出た施策に倍張りする。

## 2. 三者ディスカッション（対立点と合意）

| 論点 | 対立 | 合意（止揚の核） |
|---|---|---|
| **無料PR配信を使うか** | ①②「nofollowで価値薄・footprint懸念」 vs ③「タダで露出は取れる」 | **被リンク本線にはしない**。指名検索・サイテーション・記者リーチ（P-NEWS/PRONE）目的で**選別的に**使う |
| **自動アウトリーチの規模** | ③「自動で大量に」 vs ②「大量同一送信はスパムで無効化」 | **発見・評価・下書きは全自動／送信は人間が少量・高関連・パーソナライズ**に分離。これが費用ゼロと安全性の両立点 |
| **dofollow至上か** | ①「dofollowを取りに行く」 vs ②「言及・nofollowも価値」 | **言及回収（unlinked mention）を最優先**＝既に関連文脈で名前が出ている＝最も自然で成功率が高い |
| **何から作るか** | ①「リンカブル資産」②「構造化データ」③「分配の仕組み」 | **一次データを構造化して引用元化**＝3者の交点。DBがそれを満たす唯一の資産 |

**全会一致の結論**: 費用ゼロ・自動化・Google安全・Exit価値を**同時に満たす一点**は――
> 「**引用したくなる一次資産（DB・ツール）を作り、Claude Codeで"発見→評価→下書き"を自動化し、人間は最後の1マイルだけを担う**」。

## 3. 止揚（アウフヘーベン）＝ 統合戦略「Citable Asset Engine」

```
            ┌─────────────────────────────────────────────┐
            │   資産（Asset）：1,726自治体DB＋無料ツール    │  ← Googleが報いる Information Gain
            │   → Dataset構造化で"機械可読な引用元"に        │
            └───────────────┬─────────────────────────────┘
                            │ 引用されやすい形に整える
   ┌────────────────────────▼────────────────────────┐
   │  発見・評価・下書き（Discovery）：Claude Code 全自動 │  ← 費用ゼロ・無制限工数
   │  ・言及回収  ・リソースpage  ・壊れリンク  ・記者ネタ │
   └────────────────────────┬────────────────────────┘
                            │ 候補CSV＋パーソナライズ下書き
            ┌───────────────▼───────────────┐
            │  最後の1マイル（Human）：少量・高関連 │  ← 人間は承認＆関係構築のみ
            │  送信／自治体・協会・記者との関係     │
            └───────────────┬───────────────┘
                            │ 月次で参照ドメイン増加を計測
            ┌───────────────▼───────────────┐
            │  複利（Compounding）：四半期データドロップ │ ← Exit評価額を押し上げる成長グラフ
            └─────────────────────────────────┘
```

## 6. やってはいけない（ペナルティ／無効化 回避）
- ❌ 低品質ディレクトリへの大量登録（footprint・無価値）
- ❌ 相互リンクの組織化／リンク交換の量産
- ❌ 同一文面の大量自動送信（＝スパム判定）。**送信は人間・少量・パーソナライズ**
- ❌ 完全一致 商用アンカーの作為（「不動産査定」等を狙わない）
- ❌ 有料リンク・PBN・サテライト乱立
- ❌ コメント/フォーラム/Q&Aへの自作自演リンク投下
- ❌ 自治体への初回「リンク掲載依頼」（必ず"内容確認"先行）

## 7. 計測（KPI）
- **主KPI**: 参照ドメイン数（Referring Domains）の**月次純増**
- 補助: `.lg.jp/.go.jp/.ac.jp` 比率、言及回収の成約率、データレポートの被引用数、指名検索ボリューム、リンク経由のreferralセッション/CV
- ツール: GSC（リンク レポート）＋ 既存`links:profile`（Ahrefs/GSC CSVをinboxへ）
